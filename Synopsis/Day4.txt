1. PROMISES & ASYNCHRONOUS JAVASCRIPT
   Concept: A Promise represents a value that will be available in the future. 
   It is used to handle asynchronous operations like API calls and timers.

   A Promise has three states:
   - Pending: Initial state, operation is still running.
   - Fulfilled (Resolved): Operation completed successfully.
   - Rejected: Operation failed.

   Example:
   function getData() {
       return new Promise((resolve, reject) => {
           setTimeout(() => {
               resolve("Data received");
           }, 2000);
       });
   }

   getData()
       .then((data) => {
           console.log(data); // Runs when promise is resolved
       })
       .catch((error) => {
           console.error("Error:", error); // Runs when promise is rejected
       });

   Explanation:
   - new Promise() is used to create an asynchronous task.
   - resolve() is called when the task is successful.
   - reject() is called when the task fails.
   - .then() handles the success result.
   - .catch() handles errors.
   - setTimeout() is used here to simulate a delay like an API call.

2. TIMERS: SETTIMEOUT VS SETINTERVAL
   Concept: Controlling the timing of code execution.
   - setTimeout: Executes once after a specified delay.
   - setInterval: Executes repeatedly at a fixed time interval.
   Example:
   setTimeout(() => console.log("Once after 5s"), 5000);
   setInterval(() => console.log("Repeating every 3s"), 3000);

3. API FETCHING (FETCH API)
   Concept: Using the built-in fetch() method to request data from external APIs.
   - res.json(): Converts the raw response into a usable JS object.
   - Data Transformation: Extracting specific fields, filtering results, and 
     performing calculations.

   Practice Portfolio (10 API Examples):
   1.  Weather API: Celsius to Fahrenheit conversion.
   2.  JSONPlaceholder (Todos): Filtering completed tasks.
   3.  JSONPlaceholder (Users): Extracting nested address data.
   4.  Fake Store: Filtering by price and calculating total sum (reduce).
   5.  Random User: Mapping results to get names and countries.
   6.  Dog API: Extracting image URLs from an array.
   7.  REST Countries: Filtering by population (>100M).
   8.  Bored API: Fetching random activities.
   9.  SpaceX: Filtering successful launches.
   10. Cat Facts: Extracting string data and its length.

   Example Structure:
   fetch(url)
     .then(res => res.json())
     .then(data => {
         // Logic: Filter -> Map -> Process
         const result = data.map(item => ({ ... })); 
         console.log(result);
     });

#############################################################################
KEY TAKEAWAYS (DAY 4)
#############################################################################
- Promises handle operations that take time without freezing the browser.
- Always use .catch() to handle potential network errors in fetch() calls.
- setTimeout is for one-time delays, while setInterval is for recurring actions.
- Practice with multiple public APIs builds confidence in handling diverse JSON.
