############################################################
DAY 23: PLAYWRIGHT CI/CD PIPELINE WITH ALLURE REPORTING
############################################################

1. GITHUB ACTIONS ADVANCED WORKFLOW

   * Concept: Executes Playwright tests automatically and generates Allure reports using a multi-job CI/CD pipeline.
   * Workflow Name:

     ```yaml
     name: Playwright Tests & Allure Report
     ```
   * Triggers:

     ```yaml
     push
     pull_request
     workflow_dispatch
     ```
   * Purpose:

     * Runs tests on every code update.
     * Supports manual execution from GitHub Actions.
     * Generates and publishes test reports automatically.

2. GITHUB PERMISSIONS CONFIGURATION

   * Permissions:

     ```yaml
     contents: write
     pages: write
     id-token: write
     ```
   * Purpose:

     * Allows report deployment to GitHub Pages.
     * Grants required access for workflow execution and artifact publishing.

3. PARALLEL TEST EXECUTION (SHARDING)

   * Concept: Splits test execution into multiple shards for faster execution.
   * Configuration:

     ```yaml
     strategy:
       fail-fast: false
       matrix:
         shard_index: [1, 2, 3]
     ```
   * Execution:

     ```yaml
     npm run test:ci -- --shard=1/3
     npm run test:ci -- --shard=2/3
     npm run test:ci -- --shard=3/3
     ```
   * Benefits:

     * Reduces overall execution time.
     * Allows large test suites to run efficiently.

4. REPOSITORY CHECKOUT & NODE.JS SETUP

   * Repository Download:

     ```yaml
     uses: actions/checkout@v4
     ```
   * Node.js Setup:

     ```yaml
     uses: actions/setup-node@v4
     ```
   * Additional Feature:

     ```yaml
     cache: npm
     ```
   * Purpose:

     * Installs Node.js environment.
     * Improves workflow speed through dependency caching.

5. DEPENDENCY INSTALLATION

   * Command:

     ```yaml
     npm ci
     ```
   * Purpose:

     * Installs all project packages.
     * Ensures clean and consistent dependency management.

6. PLAYWRIGHT BROWSER INSTALLATION

   * Command:

     ```yaml
     npx playwright install --with-deps
     ```
   * Purpose:

     * Installs Chromium, Firefox, and WebKit.
     * Downloads required operating system dependencies.

7. AUTOMATED TEST EXECUTION

   * Command:

     ```yaml
     npm run test:ci
     ```
   * Additional Configuration:

     ```yaml
     continue-on-error: true
     ```
   * Purpose:

     * Executes all Playwright test cases.
     * Allows workflow continuation even if some tests fail.

8. ALLURE RAW RESULTS COLLECTION

   * Configuration:

     ```yaml
     uses: actions/upload-artifact@v4
     ```
   * Artifact Name:

     ```yaml
     allure-results-shard-1
     allure-results-shard-2
     allure-results-shard-3
     ```
   * Purpose:

     * Stores Allure result files generated by each shard.
     * Makes results available for report generation.

############################################################
REPORT GENERATION 
############################################################

9. DOWNLOAD SHARDED TEST RESULTS

   * Configuration:

     ```yaml
     uses: actions/download-artifact@v4
     ```
   * Purpose:

     * Downloads Allure results from all shards.
     * Merges results into a single directory.

10. ALLURE REPORT GENERATION

    * Command:

      ```yaml
      npm run allure:generate
      ```
    * Purpose:

      * Converts raw test results into an interactive HTML report.
      * Provides detailed test execution insights.

11. ALLURE REPORT UPLOAD

    * Artifact:

      ```yaml
      allure-report
      ```
    * Purpose:

      * Stores generated HTML reports.
      * Allows team members to download and review reports.

12. GITHUB PAGES DEPLOYMENT

    * Action:

      ```yaml
      peaceiris/actions-gh-pages@v3
      ```
    * Deployment Branch:

      ```yaml
      gh-pages
      ```
    * Purpose:

      * Publishes Allure reports online.
      * Makes reports accessible through GitHub Pages.

############################################################
KEY TAKEAWAYS (DAY 23)
############################################################

* GitHub Actions automates Playwright test execution and report generation.
* Matrix sharding enables parallel execution, reducing test run time.
* npm caching improves workflow performance and reduces installation time.
* Allure provides rich HTML reports with detailed execution insights.
* Artifacts preserve both raw results and generated reports.
* continue-on-error ensures report generation even when tests fail.
* GitHub Pages hosting allows easy sharing of test execution reports.
* The workflow follows a complete CI/CD pipeline: Test Execution → Result Collection → Report Generation → Report Deployment.
* This setup is highly suitable for large Playwright automation frameworks containing 100+ test cases.
